# 产品愿景与定位

## 一句话

**sag 是一套上下文基础设施（Context Infrastructure）。** 它把散落在文档、消息、会话、录音里的信息，持续汇聚成结构化的「上下文」，再以「灵魂」的形态提供给人和 Agent 使用。

> 一套基座，三种面孔：
> - **个人的信息助手** —— 你的第二大脑，记得你说过的、读过的、聊过的。
> - **企业的决策大脑** —— 可审计、可治理的组织知识与洞察中枢。
> - **Agent 的灵魂** —— 给任意 Agent 注入你的上下文与人格，让它「成为你熟悉的它」。

## 我们真正在做什么

不是又一个「文档问答」。RAGFlow 类产品把 RAG 做成了对静态文档的检索问答；sag 把视角提高一层：

**上下文（Context）本身才是产品。** 检索问答只是上下文的一种消费方式。sag 关注上下文的**全生命周期**：

```
获取(Acquire) → 结构化(Ingest+Extract) → 记忆(Remember) → 提供(Serve) → 拓展(Extend)
  连接器          SAG 分块/向量/事件图谱      会话记忆       灵魂检索生成     插件/挂载/应用
```

底层能力来自 `zleap-sag`（本地优先的记忆 / 知识引擎）：解析、分块、向量、**事件—实体图谱**、多策略检索。sag 在其上做成一个**有吸引力又有生产力的产品**。

## 为什么能超越 RAGFlow

| 维度 | RAGFlow 类 | sag |
|---|---|---|
| 数据来源 | 静态文档为主 | 文档 **+ 消息 + 会话 + 录音**，静态上传 **+ 动态连接器持续写入** |
| 结构 | 切块 + 向量 | 切块 + 向量 **+ 事件—实体图谱**（多跳检索） |
| 交互对象 | 知识库机器人 | **灵魂**：有名字、人格、记忆，绑定多个上下文 |
| 记忆 | 无 / 弱 | **会话即记忆**：对话自动沉淀为可检索、可抽取的上下文 |
| Agent 原生 | 无 | **MCP / Skill / Hook** 上下文插件；**挂载到本地 Agent** |
| 组织 | 库 | **命名空间（文件夹）** 组织信源；工作空间多租户 |
| 企业级 | 参差 | **审计清晰、日志完备、配置灵活、RBAC**，可直接部署 |
| 想象力 | 工具 | **上传一本书→与书中人物对话**；本地 Agent「夺舍」 |

一句话：RAGFlow 是「文档问答工具」，sag 是「**上下文操作系统**」。

## 命名体系 v2（统一语言 · 依据 [调研收敛](research-convergence.md)）

> 2026-07 收敛决定：**回归行业标准词**。sag 自己叫「AI Agent / 智能同事」，RAGFlow 叫 Knowledge base / Agent——「灵魂」等比喻词降级为愿景叙事，**绝不出现在界面**。

| 界面用词（唯一） | 内部代码 | 含义 |
|---|---|---|
| **Agent** | `Soul` | 可对话的实体：名字 + 设定 + 绑定的知识库 + 记忆 |
| **知识库** | `Source` | 一个可检索的内容集合，1:1 映射引擎 `source_config_id`；有类型（文档/网页/消息/会话/录音） |
| **分组** | `Namespace` | 知识库的文件夹（默认「会话记忆」「知识」），可整组绑定给 Agent |
| **设定** | `persona` | Agent 的角色说明、开场白、回答规则与检索参数 |
| **记忆** | conversation source | 与 Agent 的对话自动沉淀、可被再次检索（作为 Agent 的属性呈现） |
| **连接器** | `Connector` | 把外部内容持续写入知识库的适配器 |
| **挂载** | `Mount` | 把 sag 的知识与设定注入外部（本地）Agent |

> 内部标识符 / 表名 / API 路径（`Soul`、`/souls` 等）**不迁移**——避免无意义的破坏性改动；对外文档标注映射即可。「上下文 / Context Infra」只作愿景词使用。

## 设计原则

1. **从基础出发，再拓展。** 每个宏大能力都落在「上下文」这一地基上，先把地基做扎实。
2. **上下文优先。** 一切能力围绕「如何更好地获取、结构化、记忆、提供上下文」。
3. **标准接入、可拓展。** 连接器、灵魂、插件都走统一协议，新增即插入，不改上层。
4. **企业级严谨。** 审计清晰、日志完备、配置灵活、权限分明，可直接部署使用。
5. **克制而升级的审美。** 纸墨 + 淡金的克制底色之上，把细节、动效、信息密度打磨到精致。
6. **生产力而非 demo。** 每个功能都要能真正用起来、扛得住规模、经得起审计。
