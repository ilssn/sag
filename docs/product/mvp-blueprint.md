# MVP 定稿蓝图（v1 · 2026-07）

> 在 [调研收敛](research-convergence.md) 基础上的**产品定稿**。此后界面与实现以本篇为准；被砍能力的代码作为扩展点保留在后端，见文末。

## 一、概念：只有四个，一句话说清

| 概念 | 一句话 | 用户动作 |
|---|---|---|
| **信源** | 装内容的地方 | 新建 → 上传文档 → 自动解析入库 |
| **助手** | 用内容对话的 AI | 创建 → 绑定信源 → 对话（带引用、有记忆） |
| **记忆** | 助手自动记住聊过的 | 无需操作，作为助手的能力呈现 |
| **搜索** | 直接查内容原文 | ⌘K 随时呼出，搜完即回 |

- 不再有：分组、命名空间、灵魂、上下文、知识库、人格、连接器（这些词**一律不出现**在界面）。
- 「助手」= 工程里的 agent；内部代码标识符（`Soul`/`Source`/`Namespace`）不迁移。

## 二、信息架构

```
侧边导航（4 项，无更多）        全局浮层
─────────────────────         ─────────────
总览   引导 + 搜索入口 + 概况    搜索（⌘K / 点击呼出）
信源   列表 → 详情(上传/文档)      · 默认搜全部信源
助手   列表 → 工作台(对话/设定/绑定) · 从信源详情呼出时自动 @该信源 锁定
设置   模型 / 账户 / 外观          · Enter 检索 · ↑↓ 选择 · Esc 关闭返回原页
```

## 三、用户旅程（黄金路径，三步）

```
① 新建信源，上传文档        →  自动解析 · 分块 · 抽取（状态实时可见）
② 创建助手，绑定该信源      →  写个名字和设定即可
③ 开始对话                 →  回答带引用（可溯源到信源原文），越聊越有记忆
搜索贯穿始终：任何时刻 ⌘K 直查原文，不打断当前工作。
```

- **总览页职责**＝这条路径的驾驶舱：顶部搜索入口；未完成时显示三步引导；完成后显示一行概况（非卡片马赛克）+ 我的助手 + 最近信源。

## 四、搜索交互规格（本次新增的唯一能力）

- **呼出**：全局 `⌘K / Ctrl+K`；总览顶部的搜索框样入口；信源详情的「搜索」按钮（此时带 `@信源名` 范围芯片，可删除芯片扩大到全部）。
- **检索**：Enter 触发（向量检索有延迟，不做输入即搜）；加载态 spinner；结果为段落卡：**信源名徽标 + 标题 + 片段 + 分数**。
- **键盘**：`↑↓` 移动选中，`Enter` 打开所属信源详情，`Esc` 关闭——浮层关闭即回到原页面，零跳转成本。
- **后端**：新增 `POST /api/v1/search`（工作空间级，多信源 fan-out，可传 `source_ids` 收窄），复用 `EngineManager.search_many`，结果带 `source_name`。

## 五、本次砍掉的界面（代码去向）

| 砍掉 | 原因 | 代码去向 |
|---|---|---|
| 单信源问答页 `/ask` | 对话只有一个入口＝助手；快查交给搜索 | 前后端一并移除（源级 chat API/服务/页面），概念唯一 |
| 分组（命名空间）UI | 概念税；MVP 信源量小用不上 | 后端 `Namespace` 保留（记忆内部依赖），UI 全部移除 |
| 连接器选择（网页同步） | 暂不做连接器；新建=只有上传 | 后端连接器框架与 web 实现保留；创建对话框只走上传 |
| 实体洞察 / 提取成助手 | 暂不做人格提取 | 后端 insights API 保留（实验性）；前端组件删除 |

## 六、扩展点（都在，只是不见于界面）

- `connectors/`：注册表 + web 连接器完整可用 → 未来「接入更多来源」直接开闸。
- `namespaces`：模型与默认播种仍在 → 未来信源多了再开「分组」。
- `insights`（实体/热度/`to-soul`）：API 与测试仍在 → 未来「书→人物」作为彩蛋回归。
- `search_many` / 记忆闭环 / SSE 流式：均为地基，长期不变。

## 七、质量门（不变）

`pytest` 全绿 · `ruff` 全绿 · `next build` 全绿；每屏四态齐全（载/空/错/满）；文案=动作本身；淡金仅作强调；动效仅两处。
