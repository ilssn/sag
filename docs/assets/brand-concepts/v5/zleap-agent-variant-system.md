# Zleap 多 Agent 视觉变量规范 v1

## 1. 系统原则

Zleap 多 Agent 采用“一个母版、六个受控变量”的设计方式。

- 母版不变量：头盔、屏幕外框、耳罩、身体骨架、腿脚、胸牌、接口位置。
- 受控变量：角色配色、屏幕显示层、待机表情、Mini 天线、背部一体机、微推进器。
- 每个 Agent 最多增加一个一级插件；其余专业差异通过屏幕和背部一体机表达。
- Normal 是完整身体且不显示天线；Mini 只保留同一颗头，并显示对应天线。

## 2. 公共母版

### 色彩与材质

- 外壳：瓷白 `#F7F6F2`。
- 次级面板：暖浅灰 `#E9E8E4`。
- 关节与鞋底：石墨灰 `#4D5861`。
- 品牌金：`#CFA34B`，仅用于胸牌细边、耳罩内环和定位线，可见面积不超过 3%。
- 非屏幕机身至少 70% 保持白色；角色色在非屏幕硬件上的占比不超过 8%。
- 外壳使用细腻半哑光材质；屏幕使用高光或缎面玻璃。

### Normal / Mini

- Normal：完整 Q 版身体，胸前统一显示 `Zleap`，默认不显示天线。
- Mini：只保留与 Normal 完全相同的头盔、屏幕和耳罩；不得出现身体、背包、手脚或颈部残段。
- Mini 天线统一从头盔右后上方的标准接口伸出，只允许改变杆形和端点。

### 胸牌

- 文案固定为 `Zleap`，大小写不可变化，不加角色名或问号。
- 胸牌宽为躯干宽的 42%–46%，宽高比约 2.1:1。
- 深海军蓝几何无衬线字，瓷白底，品牌金细边。

## 4. 屏幕语言

屏幕始终分为三层，不能把功能符号直接当作永久角色脸。

1. 固定层：屏幕外框、圆角和镜片轮廓完全一致。
2. 身份层：角色专属底色、玻璃材质和低对比纹理。
3. 状态层：待机脸或工作 UI；任务结束后必须恢复默认待机脸。

表情位置统一：

- 眼睛基线位于屏幕高度的 44% ± 2%。
- 两眼中心距为屏幕宽度的 30% ± 3%。
- 嘴中心位于屏幕高度的 66% ± 2%。
- 表情笔画粗细为屏幕高度的 3%–4%，全部使用圆头笔画。

## 5. 尺寸与占比

以 Normal 站立总高 `H = 100`、头宽 `Wh`、头高 `Hh` 为基准：

- 头高：0.45–0.47H；头宽：0.54–0.57H。
- 屏幕宽：0.72–0.75Wh；屏幕高：0.56–0.60Hh。
- 肩部最外宽：0.70–0.74Wh。
- 肚子最大宽：0.60–0.64Wh。
- 躯干高：0.28–0.30H；髋到脚底：0.24–0.27H。
- 中立站姿两脚等长误差不超过 2%，脚底共面，脚尖外撇 0°–4°。
- 背部一体机正视不得超出肩部最外轮廓；顶部不得高于耳罩下缘；底部止于髋线附近。
- 微推进器固定为两个；视觉面积不超过完整轮廓的 6%；运动尾迹最长不超过 0.35H。
- Mini 天线高度控制在 0.20–0.25Hh，外扩不超过头宽的 15%。

背部一体机尺寸档位：

- XS：约 `0.46Wh × 0.48Hh × 0.14Wh`。
- S：约 `0.50Wh × 0.52Hh × 0.15Wh`。
- M：约 `0.56–0.58Wh × 0.60–0.61Hh × 0.18–0.19Wh`。
- L：约 `0.62Wh × 0.68Hh × 0.21Wh`。

## 6. 推进器规则

- 所有推进尾迹统一使用蓝白冷光；角色色只用于喷口状态环。
- 待机时只显示薄光环，不显示尾迹。
- 禁止大型外露涡轮、火箭筒、火焰、厚重喷管或独立外挂发动机。
- 差异来自喷口几何、脉冲节奏和尾迹形态，不来自体积堆叠。

## 7. 禁用项

- 只换配色，不改变专业组件语义。
- 改变头盔、屏幕外框、身体骨架、腿脚或胸牌位置。
- Normal 常驻天线；Mini 携带身体或背部一体机。
- 把 `01`、`∞`、`>_` 永久作为角色待机表情。
- 背包袋、服务器堆栈、外挂火箭罐、裸露线缆或重装甲。
- 每个 Agent 同时使用多个一级插件，造成轮廓混乱。
- 高饱和霓虹、彩色尾焰、武器、尖刺或军用符号。
