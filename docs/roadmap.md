# 路线图

> 原则：**从基础出发，再拓展**；每阶段都是可用的纵切、有验收、并复用既有地基。每阶段末尾都要求「UI 精致 + 可验证」。

## Phase 0 — 知识库地基 ✅（已完成 P0–P4a）
多用户 · 信源(1:1 引擎) · 文档上传 · 后台处理(ingest→extract) · 单源检索 · 流式问答+引用 · Web 连接器同步 · docker compose · 纸墨+淡金 UI。
**已验证**：后端端到端 + 前端构建 + Web 同步离线 e2e。

## Phase 1 — 组织与配置
**目标**：把「库」升级为「有结构的上下文」，并让配置可视化。
- 数据：`namespaces` + `sources.namespace_id/source_type`；默认「会话记忆/知识」；迁移回填。
- UI：导航加「上下文」（命名空间树）；信源带类型徽标。
- 可视化模型配置（设置页，热应用，替代 .env）。
**验收**：建命名空间、移动信源、改模型即时生效。**复用**：现信源/文档流。

## Phase 2 — 灵魂（旗舰）
**目标**：把「问答」升华为「和有人格的灵魂对话」。
- 数据：`souls` + `persona_json` + `soul_bindings`。
- 引擎：`sag/` 新增 **多信源 fan-out** `search_many` + 合并/重排。
- UI：灵魂列表 + **灵魂工作台**（对话 + 人格编辑 + 绑定管理）；问答迁入。
**验收**：建灵魂→绑定多信源→跨源带引用问答。**复用**：SSE 生成、引用、检索。

## Phase 3 — 记忆闭环
**目标**：越聊越懂你。
- 会话 → `conversation` 信源（落灵魂会话记忆空间）→ 自动 `extract`。
- 检索 targets 纳入记忆；隐私开关（仅本人/全部）；记忆治理（摘要/TTL/遗忘）。
**验收**：聊过的内容下次能被检索引用。**复用**：ingest/extract 管线、fan-out。

## Phase 4 — 持续接入
**目标**：上下文源源不断。
- 统一 Ingestion API：`/sources/{id}/messages` `/ingest` + 标准消息格式 + 缓冲成块。
- 连接器扩展：`message_api`、`notion`、`feishu`、`audio`（上传转写 ASR 前置处理器）。
- 写入令牌（作用域化、可吊销、进审计）。
**验收**：外部系统持续推消息→自动成块入库→可被灵魂检索。**复用**：Connector 协议、注册表、process_document。

## Phase 5 — 插件与挂载
**目标**：让任意 Agent 拥有你的上下文与灵魂。
- MCP Server（`search_context/get_entity/remember/…`）+ Skill/Hook 打包。
- 挂载包 + `MountAdapter`（Claude Code/Cursor/通用）；知识挂载 & 全量夺舍（备份/写入/还原/提示）。
- UI：挂载向导。
**验收**：本地 Agent 知识挂载可引用私有知识；夺舍后变灵魂、可一键还原。**复用**：fan-out=search_context。

## Phase 6 — 洞察与想象力
**目标**：把图谱变成明星功能。
- 洞察页：实体分类 + 人物热度榜 + 实体详情。
- **书→人物**：实体→灵魂（LLM 生成人格，绑定书信源）；与书中人物对话。
- 检索调试可视化。
**验收**：上传一本书→人物榜→一键成灵魂→带书中情节引用对话。**复用**：SAG 图谱、souls、人格生成。

## Phase 7 — 企业级硬化
**目标**：可直接进企业。
- 审计（不可变 + 查询/导出）、结构化日志 + trace、任务/指标面板。
- RBAC 细化、密钥加密存储与吊销、彻底删除（合规）。
- 部署：K8s/Helm、对象存储、PG 备份/PITR、区域化。
**验收**：全动作可审计导出；权限隔离；可扩容部署。**复用**：现 logging/错误映射/compose。

---

## 横切（每阶段都做）
- **设计系统成熟度**：每阶段交付**精致可用**的 UI，不留半成品；补齐组件（⌘K、Tree、抽屉、Chart、Persona 编辑器…）。
- **文档**：功能随文档走（本 docs/ 持续更新）。
- **验证**：每阶段端到端验证 + 关键单测；`dev` 分支多次提交，阶段稳定后合 `main`。

## 依赖顺序（关键路径）
```
P1 组织/配置 ─→ P2 灵魂(fan-out) ─→ P3 记忆
                     └─→ P4 持续接入 ─→ P5 挂载
                     └─→ P6 洞察/书→人物
P7 企业硬化：与 P2 起并行推进（审计/日志随每个新写操作补齐）
```
建议先打通 **P1→P2→P3**（组织→灵魂→记忆）这条主线，它解锁最多场景且最能体现「上下文基座」的差异化。
