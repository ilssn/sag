# 设计系统与信息架构

> 审美要升级，但方向不变：**纸墨极简 + 淡金**的克制底色之上，把层次、动效、信息密度打磨到精致。克制不等于平淡——靠细节取胜。

## 一、设计系统升级

现有令牌（`app/globals.css`）已定义纸墨 + 淡金、亮暗双主题。升级方向：

### 层次与材质
- **三级表面**：`paper`（页底）→ `surface`（卡片）→ `surface-2`（内嵌/激活），已具雏形；再补 `elevation`（浮层/弹窗用更强阴影 + 极淡描边）。
- 分隔靠 **1px 发丝线 + 留白** 而非重色块；淡金**只**用于强调（激活/焦点/主 CTA/引用/进度），坚持「不铺金」。
- 卡片圆角 10–12、hover 微升 2px + 阴影渐强（已实现于信源卡）。

### 字体与排版
- 标题衬线 Fraunces（灵动）+ 正文 Inter + 等宽 JetBrains Mono（已接）。
- 固定**字阶**（如 12/13/14/16/20/24/30）并严守；标题 `text-wrap: balance`；正文≈65 字宽。
- 数字统一 `tabular-nums`（统计/时间线）。

### 动效语言
- 时长 150–240ms，`ease-smooth`（已定义）；仅用于**状态过渡**，不炫技。
- 规范化：进入 `fade-in`（4px 上移）、流式 `blink` 光标、骨架 `shimmer`（已有）。
- 全站尊重 `prefers-reduced-motion`。
- 「灵魂」相关可有**克制的性格化微动效**（如灵魂在线的呼吸态圆点），但一处点睛即可。

### 状态与空态（生产级细节）
每个数据视图四态齐全：**加载(骨架) / 空(引导+主动作) / 错(可重试+说明) / 满(内容)**。已在信源/文档/问答落地，需推广到灵魂/连接器/洞察。文案「从用户侧」写，动作即结果（Publish→Published）。

### 无障碍
焦点环恒可见（淡金，已实现）；淡金对比不足处用 `gold-strong`/加粗；键盘可达；语义化。

## 二、信息架构（导航演进）

当前：概览 / 信源 / 问答 / 设置。演进为围绕「灵魂 + 上下文」的五区：

```
主页 · 概览        今日、最近灵魂/活动、待处理
灵魂 Souls        灵魂列表 → 灵魂工作台（对话 + 人格 + 绑定）   ← 问答归入这里
上下文 Context     命名空间树 + 信源；信源详情（上传/同步/文档/状态）
连接器 Connectors  连接器画廊 + 已接入实例 + 同步状态
洞察 Insights      实体/事件图谱、人物热度、书→人物、检索调试
设置 Settings      模型与检索(可视化) · 成员/权限 · 密钥 · 审计 · 外观
```

> 迁移平滑：现「问答」→「灵魂工作台」；现「信源」→「上下文」（加命名空间层）。⌘K 命令面板贯穿全局（跳转/新建/搜索）。

## 三、关键界面规格

### 灵魂工作台（旗舰）
```
┌ 灵魂头像 + 名字 + 在线态 ┈┈┈┈┈┈┈┈┈┈┈┈┈┈ [人格] [绑定] [挂载] ┐
│ 左：会话列表(新会话/历史)                                        │
│ 中：对话流（用户右/灵魂左 · Markdown · [n]引用可展开来源卡）      │
│ 右(可折叠)：本轮引用的上下文 + 命中的实体/事件（可点进洞察）      │
│ 底：输入框（Enter 发送）                                          │
└ 侧抽屉：人格编辑（system_prompt/语气/traits/检索参数）· 绑定管理 ┘
```
情感锚点：灵魂头像与名字始终在场；「记得你」通过引用其**会话记忆**来可视化。

### 上下文库
- 左：命名空间树（会话记忆/知识/自定义，可增删）。
- 右：该空间的信源卡（类型徽标：文档/消息/会话/录音/网页）；进入信源 = 现详情页（上传/同步/文档/时间线）。

### 连接器画廊
- 卡片网格：每个连接器的图标/标题/描述/「可同步」徽标（元数据已由后端提供，表单已能动态渲染）。
- 接入向导：选连接器 → 动态表单填配置 → 建信源 → 首次同步。

### 洞察
- 实体列表（按类型分组）+ **人物热度榜**（频次×中心度）；实体详情（相关事件/来源）。
- **书→人物**：某文档信源上「一键把人物提成灵魂」。
- 检索调试：对信源/灵魂跑 query，看命中段落与分数（`/search` 已具后端）。

### 挂载向导
- 选灵魂 → 选宿主(Claude Code/Cursor/通用 MCP) → 选模式(知识/夺舍) → 生成命令或配置 → 复制即用；full 模式明确提示「将备份旧人格，可一键还原」。

### 设置 · 可视化配置
- 模型与检索：表单填 LLM/embedding/base_url/key/策略，保存即热应用（替代改 .env）。
- 成员/权限、密钥、**审计日志查询/导出**、外观（亮暗/密度）。

## 四、组件库成熟度

在现有 shadcn 定制基础上补齐：命令面板(⌘K, cmdk)、Tabs、Tooltip 普及、Sheet/抽屉、Tree（命名空间）、数据表（审计/文档）、Chart（洞察/指标）、Persona 编辑器、Binding 选择器、连接器卡、灵魂头像（可生成）。全部走既有令牌，保证「一眼是 muse」。

> 审美升级 = 更强的层次 + 更规范的动效 + 更完备的状态 + 更贴切的信息架构，而非更多颜色或装饰。
