# Tauri 与 sidecar 使用结构化启动协议

Tauri 启动 FastAPI sidecar 时提供数据目录、稳定端口和一次性 nonce；sidecar 先占用端口，再以机器可读事件依次报告启动、迁移、引擎初始化、就绪或错误。就绪事件必须回传 nonce、API 版本和能力列表，Tauri 验证后才向静态前端提供运行时配置。该协议替代固定延时和对端口的盲目轮询，既能展示准确启动阶段，也能避免把占用同一端口的其他进程误认为 SAG。sidecar 在就绪后意外退出时最多自动重启三次并使用退避等待，后台任务从持久检查点恢复；端口冲突、迁移失败、数据版本不兼容、显式退出或连续失败不进入无限重启。
