# 使用焦点关系片与分态布局

知识宇宙不再用一套“冻结力导向 + 横向散点 + 卡片配额/碰撞”同时承担时空探索和线索整理。时空探索态采用确定性的窄幅时间走廊：窗口内全部事件、实体和真实关系继续作为星体网络存在，时间序只推进纵深，稳定哈希只提供轻微横向错落，相关实体使用事件本地的确定性径向槽位。相机深度在中央安全视区自然形成约 3–5 个连续事件的焦点关系片，滚轮按事件连续推进、底部按钮按焦点关系片推进，网络分页只补充缓存；普通实体保持为星体，hover 或锁定才完整展开当前一跳实体卡片，且不移动布局或相机。焦点片内部与共享实体桥清晰显示，其他关系作为低显著度背景脉络。线索累积态采用独立的增量关系布局：仅新增批次在中央前景短暂收敛并桥接旧图，既有节点不重排，沉降完成后冻结，相机只跟随新增批次与最近桥接邻居，超过窗口上限后按事件 FIFO 渐隐移除。普通点击只锁定关系焦点，详情必须通过显式入口打开。该决策取代 ADR 0023 的固定卡片预览/阅读孔径，并取代 ADR 0024、0026 中关于“预览 10 张事件卡片”的部分；不保留旧布局或偏好兼容路径。
