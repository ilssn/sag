# 桌面主面板与伴随宠物互斥显示

该方案适用于首版之后的桌面伴随里程碑，不属于桌面 V1。届时桌面客户端采用互斥的主面板形态与桌面伴随形态：主面板可见时，宠物始终渲染在主面板内，只能缩小而不能关闭；桌面专属的“宠物常驻”偏好只决定主面板退场后是否保留桌面宠物，且默认开启。开启宠物常驻后，最小化或关闭主面板都会让主面板退出，仅显示可以唤出 Mini 面板的桌面宠物，只有托盘、宠物右键菜单或应用菜单中的明确“退出 SAG”动作才结束进程；关闭宠物常驻后，最小化恢复系统标准行为，关闭主面板则退出进程。选择该模型而非让主面板与独立宠物同时存在，是为了保持主面板与 Web 体验一致，并避免双宠物、跨窗口焦点竞争、重复交互状态以及误关正在运行的任务。
