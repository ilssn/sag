# SAG

SAG 是面向个人与 Agent 的知识库应用。品牌愿景是成为用户管理、检索和复用知识时需要的最后一个知识库应用。

## Language

**知识库应用（Knowledge Base Application）**:
用户直接使用的完整产品，包含知识导入、组织、检索、溯源、对话与复用能力。SAG 的品牌表达是“你的最后一个知识库应用”。
_Avoid_: 用“知识库”单独指代完整产品

**知识库（Knowledge Base）**:
知识库应用内由信源、文档及其结构化索引组成的知识集合，不等同于应用本身。
_Avoid_: 应用、产品

**知识宇宙（Knowledge Universe）**:
知识库中事件、实体和事实关系的有界 4D 探索空间，以三维空间表达关系与纵深，以时间或叙事顺序表达演进；它是同一事实库的交互投影，不是第二份知识内容。
_Avoid_: 全库图、第二知识库、独立知识图谱

**信息源探索会话（Source Browse Session）**:
用户在单一信息源快照内按时间或叙事顺序连续探索事实的会话；切换信息源即开始新会话。
_Avoid_: 搜索会话、全库会话、跨信息源缓存

**时空探索态（Temporal Exploration State）**:
知识宇宙中沿单一信息源的时间或叙事顺序持续深入的工作状态，当前可见事实随观察位置呈现明确的远近与先后；推进由用户或显式自动播放掌控，停稳后观察位置保持稳定。场景采用确定性的时间走廊和局部关系簇布局，不运行全局力导向或碰撞模拟。
_Avoid_: 普通模式、预览模式、时间翻页模式

**线索累积态（Evidence Accumulation State）**:
知识宇宙中临时汇聚搜索、问答与主动拓展所得事实并阅读其关系的稳定工作状态；清除当前焦点不会退出该状态，显式退出后恢复进入前的时空探索现场。场景采用关系力导向布局：新增节点短暂参与增量收敛，既有节点保持位置，过渡停稳后冻结。
_Avoid_: 问答模式、搜索模式、永久图谱

**线索图谱（Evidence Graph）**:
线索累积态中由多轮检索与主动拓展逐步形成的临时关系投影；每批新增线索增量进入当前阅读前景，窗口满后按先进先出移除最旧事项，未退出窗口的既有节点保持稳定。
_Avoid_: 永久知识图谱、每轮重排的力导向图、静默后台追加

**线索追加（Evidence Append）**:
搜索、问答或用户主动“探索更多”返回一批事件与实体后，经过身份去重和关系合并再提交给线索图谱的显式增量操作；线索累积态不会以预取为由检索尚未产生的新知识。去重后的新增批次进入中央阅读前景并短暂标记，通过共享实体桥接既有图谱；旧节点不重新排布，相机只轻移到新增批次及其最近桥接关系。
_Avoid_: 后台自动扩展、逐节点刷新、时间线预取

**知识宇宙窗口（Knowledge Universe Window）**:
当前场景最多参与布局和渲染的事件集合，相关实体随事件完整进入；时空探索态与线索累积态共同受同一个窗口上限约束，窗口之外的数据不进入当前场景。窗口中的事件全部以星体参与关系网络，但不等于全部同时展开为完整卡片。
_Avoid_: 请求分页、完整卡片数量、缓存容量

**增量窗口过渡（Incremental Window Transition）**:
新事项从缓存按顺序进入知识宇宙窗口、等量最旧事项按先进先出在边缘渐隐退出的连续视觉提交；网络分页只补充缓存，不触发整页替换，过渡完成后窗口恢复配置上限。
_Avoid_: 整页闪切、全场景重建、逐节点网络请求

**事件缓存（Event Cache）**:
知识宇宙会话内部保存已加载事件、相关实体与关系的纯数据 FIFO 列表，默认容纳 1000 个事件并允许用户调整容量；达到上限后按进入缓存的先后淘汰最旧记录，不使用访问热度、关系权重或桥接保护。它不创建场景节点、DOM 卡片或 GPU 资源。
_Avoid_: 知识宇宙窗口、场景工作集、已渲染节点数量

**焦点关系片（Focused Relation Slice）**:
时空探索态中位于镜头中央安全视区、当前可完整阅读的连续时间片；默认根据视口与关系密度稳定容纳约 3–5 个事件及其关联实体。知识宇宙窗口中的其余事件仍以星体和连线维持时间纵深与网络连续性，滚动使相邻时间片平滑接管中央。悬停或锁定可以临时展开一个节点的关系网络，但不改变窗口或时间位置。焦点关系片内部的事件—实体关系以及共享实体形成的跨事件桥清晰可见，通向其余窗口星体的关系保持低显著度并在聚焦时增强。
_Avoid_: 固定卡片配额、窗口内全量展开、屏幕边缘卡片、互不相关的平级卡片散点

**关系显著性层级（Relation Salience Hierarchy）**:
知识宇宙窗口中的有效关系始终保留，但视觉强度按阅读上下文分层：焦点关系片内部与共享实体桥为主要关系，通向非焦点星体的关系为背景脉络，悬停或锁定时仅提升当前节点的一跳网络。层级只改变表现，不删除关系数据。
_Avoid_: 全窗口等亮连线、以粗线替代关系层级、聚焦时重建关系

**关系焦点（Relation Focus）**:
用户悬停事件或实体时临时显现该节点的一跳事件—实体卡片并增强相关连线，不改变布局或相机；点击只将同一关系网络锁定，并在主卡片上显现“查看详情”“探索更多”和“AI 问答”。再次点击同一节点、点击场景空白或在时空探索态继续滚动会解除锁定。
_Avoid_: 悬停重排、普通点击直接打开面板、事件与实体重复显示、点击空白刷新场景

**节点详情工作区（Node Detail Workspace）**:
仅由关系焦点主卡片上的“查看详情”显式打开的 mini 工作区，用于阅读事件详情、原文证据及相邻事件；关闭工作区不删除节点或改变当前时间位置。“探索更多”和“AI 问答”不复用详情打开动作，而是将结果增量追加到线索累积态。
_Avoid_: 普通节点点击自动弹窗、左侧重复详情面板、关闭详情即退出探索

**时间走廊布局（Temporal Corridor Layout）**:
时空探索态使用的确定性布局。事件按时间或叙事顺序分布于观察轴，焦点关系片稳定占据中央安全视区，相关实体围绕事件形成局部关系簇，共享实体作为相邻事件之间的桥；滚动只连续改变观察位置和焦点时间片，不触发全局重新构图。
_Avoid_: 全局力导向、随机散点、每次分页重排、用碰撞计算选择可见卡片

**时空探索手势（Temporal Exploration Gestures）**:
时空探索态中的滚轮只沿时间走廊推进或回退，不同时触发相机缩放；拖动允许在受控范围内环视当前焦点关系片，松手后相机平滑回正时间轴。线索累积态恢复常规三维操作：拖动旋转，滚轮缩放。
_Avoid_: 同一滚轮同时缩放和翻页、拖动切换工作状态、探索态无限制偏离时间轴

**累积关系布局（Accumulated Relation Layout）**:
线索累积态使用的增量力导向布局。新增事件与实体从中央阅读前景进入并短暂参与关系收敛，既有节点坐标在追加期间保持稳定，新增批次停稳后整体冻结；相机只跟随新增批次及其最近桥接关系，不重新适配整张图。窗口超过上限时，最旧事件从边缘渐隐后按先进先出移除，其不再被引用的实体随之移除。
_Avoid_: 每轮问答全图重算、沿用时间走廊坐标、整图自动缩放、永不停止的力模拟

**时序分页（Temporal Page）**:
时空探索态按有序游标从信息源一次加载的事件批次，默认每页 20 个事件；分页大小只影响网络请求和内部缓存，不决定同屏事件、卡片数量或视觉推进步长，网络页边界不呈现给用户。
_Avoid_: 知识宇宙窗口、线索追加批次、视觉翻页

**时序预取（Temporal Prefetch）**:
时空探索态为保证连续前进与回退，提前从信息源加载当前窗口前后各至多配置页数的时序分页并写入事件缓存；调度按离当前窗口由近到远串行执行，同一时刻只有一个请求，线索累积态不执行时序预取。
_Avoid_: 线索追加、搜索、问答、用户可见缓存设置

**焦点片推进（Focus Slice Advance）**:
时空探索态的用户可见推进粒度。滚轮沿事件顺序连续移动，每经过一个事件时焦点关系片滑动一格；底部前进或后退按钮一次移动一个完整焦点关系片，约 3–5 个事件。推进只消费已加载缓存，接近缓存边界时由时序预取补充，不暴露网络分页边界。
_Avoid_: 每个事件一次网络请求、按钮替换整张网络页、分页到达后全场景刷新

**知识宇宙配置（Knowledge Universe Configuration）**:
时空探索态与线索累积态共同使用的一套事件级配置：默认缓存 1000 个事项、窗口 50 个事项，并启用事项与实体卡片；时空探索默认每页 20 个事项并前后各预取 3 页。焦点关系片由视口和关系密度自适应形成，不作为第二个卡片数量配置；状态切换不切换或复制配置档案。
_Avoid_: 探索配置、累积配置、按模式维护两套参数

**SAG**:
用户直接使用的完整知识库应用，也是本仓库交付的产品。
_Avoid_: 用 SAG 单独指代论文方法或 Python 包

**SAG 检索架构（SAG Retrieval Architecture）**:
SAG 论文原创提出的 event-entity 索引与查询时动态超边检索方法。它不是传统 RAG 与 GraphRAG 的组合、封装或双路召回。
_Avoid_: SAG 应用、zleap-sag、RAG 与 GraphRAG 的融合方案

**统一检索管线（Unified Retrieval Pipeline）**:
SAG 检索架构以原创的 event-entity 索引和执行机制，同时提供语义相似性检索与关系推理能力，替代在传统 RAG 与 GraphRAG 之间选型或部署两套系统再拼接结果的做法。
_Avoid_: 双 RAG、传统 RAG 与 GraphRAG 的融合或末端拼接

**zleap-sag**:
实现 SAG 检索架构并向其他应用提供导入、抽取和检索能力的 Python 引擎。
_Avoid_: SAG 应用

**SAG 自托管 API（SAG Self-hosted API）**:
SAG 应用随 FastAPI 后端提供的 HTTP、OpenAI 兼容和 MCP 接口。开发者在自己的服务器上运行 SAG 后使用这些接口连接自定义前端或外部 Agent；它不是由项目方托管的公共云 API。
_Avoid_: zleap-sag Python API、公共云 API

**本机集成端点（Local Integration Endpoint）**:
桌面版向同一台电脑上的外部宿主提供 API 与 MCP 的稳定入口；主机固定为 `127.0.0.1`，MCP 路径固定为 `/mcp/`，端口采用公开默认值并允许用户修改和持久化。端口冲突必须显式修复，不能静默改变外部宿主已保存的地址。
_Avoid_: 客户端内部 API、每次启动随机地址、可任意填写的完整 URL

**本机访问密钥（Local Access Key）**:
桌面版首次安装时自动生成、供同一台电脑上的外部 API/MCP 宿主共用的单一长期密钥；默认访问当前用户的整个知识库，不按宿主、信源或有效期拆分，用户只能复制或重新生成它。
_Avoid_: 登录令牌、个人访问令牌列表、细粒度权限系统

**模型凭据（Model Credentials）**:
用户为生成、向量化和文档解析服务提供的 API Key；V1 在 Web 与桌面中沿用统一的后端设置存储，接口和日志不得回传正文，完整数据备份会包含这些凭据。
_Avoid_: 本机访问密钥、界面可读取的普通设置

**应用数据目录（Application Data Directory）**:
桌面版用于持久保存数据库、知识索引和导入文件的系统标准本机目录，由稳定的应用标识定位；用户可以查看或受控迁移该目录。它不是用户主目录下的 `.sag`，也不包含可删除的缓存或日志。
_Avoid_: 工作目录、`.sag`、缓存目录、用户导出目录

**桌面运行配置（Desktop Runtime Profile）**:
客户端内置 FastAPI 使用的受约束运行配置，固定采用 SQLite、LanceDB 和应用数据目录以实现零基础设施启动；它与可配置外部存储的自托管运行配置共用业务代码。
_Avoid_: 桌面专用业务后端、前端构建变量、让客户端用户选择数据库基础设施

**存储键（Storage Key）**:
数据库中用于定位应用托管文件的相对标识，由统一存储层在当前应用数据目录下解析；数据目录迁移或跨平台恢复不会改变存储键。
_Avoid_: 绝对文件路径、用户提供的任意路径

**数据库迁移（Database Migration）**:
客户端启动业务服务前执行的版本化元数据库升级；迁移前创建恢复点，成功后 sidecar 才进入就绪状态，失败时保留原数据并进入恢复流程。
_Avoid_: `create_all`、启动时临时补列、静默忽略迁移失败

**知识索引数据（Knowledge Index Data）**:
zleap-sag 生成并持久保存的向量、事件、实体及关系索引；它随应用数据版本化迁移，不能因升级不兼容而被静默删除或自动重新抽取。
_Avoid_: 缓存、可随时重建的数据

**桌面主实例（Desktop Primary Instance）**:
每个系统用户唯一拥有桌面窗口、应用数据锁、FastAPI sidecar 和本机集成端点的 SAG 进程；后续启动请求只唤醒该实例。
_Avoid_: 每个窗口一个后端、多个客户端共享活动数据目录

**Sidecar 启动协议（Sidecar Startup Protocol）**:
Tauri 与 FastAPI sidecar 之间带一次性 nonce 的结构化启动状态流；sidecar 依次报告启动、迁移、初始化、就绪或错误，只有验证就绪事件后前端才获得运行时配置。
_Avoid_: 固定等待时间、只按端口是否可访问判断就绪
